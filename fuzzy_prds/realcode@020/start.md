## Product Requirement Document

We need a single cross-platform command-line tool that wraps a set of common shell utilities — copying files, removing files, listing directory contents, printing text, searching for patterns, changing permissions, testing whether a file exists, and a few others — behind one consistent front end. Right now our build scripts break on Windows because the underlying utilities differ across platforms. The tool should take a command name as the first argument, run the corresponding utility, and report exit status, standard output, and standard error in a uniform format so it can be embedded in any build pipeline. When something goes wrong at the routing level (no command supplied, an unrecognized command, or a command that the tool explicitly refuses to support — handle those the same way the blacklist module does) it should fail with a dedicated status code distinct from the underlying utility's own failure codes. There should be a global silence flag that suppresses error output without hiding the exit status. We also need the tool to support piping input into commands that accept it from a stream, and a stream-substitution command that accepts the familiar s/pattern/replacement/ expression syntax including escaped slashes, global replacement, and an option to rewrite the source file in place. The tool should also load optional extension modules listed in a configuration file in the working directory, and raise distinct errors if that configuration file cannot be parsed or if a listed module cannot be found.
