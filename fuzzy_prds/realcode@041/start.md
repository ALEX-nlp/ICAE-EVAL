## Product Requirement Document

Hey team, we need to build out the session token auth service we've been discussing. Basically it should handle the full login/logout lifecycle for our API — when someone logs in with valid credentials they get back a token they can use on protected endpoints, and we need the usual stuff like being able to log out just the current session or nuke all sessions at once. The token should have an expiry (we talked about this in the standup, same approach as that credential refresh logic from the old auth module) and there should be a way to auto-extend sessions when someone is actively using the API. We also need to make sure stale or tampered tokens get rejected cleanly through the HTTP layer. Oh and one thing that's been a pain — users sometimes get stuck in a weird state when they have too many active sessions, so we need some kind of cap on that. The stored tokens need some lookup metadata that lines up with what was actually submitted. Also there's some config around token prefixes that needs validation before it blows up at runtime. Broadly speaking the HTTP responses need to be consistent and predictable across all these flows. Let me know if anything is unclear — I know I'm probably missing some details around the exact formats and limits but just check what we've done before.
