## Product Requirement Document

Hey team, we need to build out that bulk insert helper we've been talking about. Basically the idea is devs can queue up a bunch of rows and flush them to the database in one go instead of doing one insert at a time — that's been killing perf on the import flows. It should work kind of like how we handled the batching logic in the old data pipeline tool, where you configure a ceiling and it auto-flushes when you hit it. Rows can come in either as ordered value lists or as named field maps, both should work. There needs to be some way to get a SQL preview for different database backends without actually hitting the db — important for the QA team. Also need lifecycle hooks so teams can filter or modify rows before they land. Oh and the timestamp stuff should just work automatically, we've had too many bugs where people forget to set created_at. The writable columns list should exclude the auto-generated primary key. One thing I'm not totally sure about is the exact conflict resolution behavior per backend — I think MySQL, Postgres, and SQLite each do it differently so we'll need to handle all three. The block-style entry point should also be supported for teams that prefer that pattern. Default batch ceiling should match what we agreed on in the architecture review.
