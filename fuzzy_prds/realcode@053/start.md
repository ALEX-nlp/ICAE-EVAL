## Product Requirement Document

Hey team, we need a little utility that auto-generates those module dependency diagrams and slaps them into the right spot in our markdown docs. Right now devs are hand-drawing these things and they go stale immediately whenever someone adds or removes a dependency — it's a real pain and the docs are always lying to us. The idea is: you feed it a description of which modules depend on which, pick some visual options, and it spits out a ready-to-commit diagram. Similar vibe to what we did with the config-injection approach in the reporting pipeline last quarter, if you remember that. It should handle different colour schemes including a fully customisable one where you can pass in your own colour values. There should also be a way to zoom in on just a subset of modules — like if I only care about one module I can filter to just the stuff touching it, with those nodes visually called out somehow. If I give a filter that doesn't match anything that should fail gracefully, not blow up with a stack trace. Docs might not exist yet, or might already have a section from a previous run — both cases need to work cleanly without nuking unrelated content. Needs to be well-structured code, not one big file. Target language is Kotlin or Python, whatever the team thinks fits best.
