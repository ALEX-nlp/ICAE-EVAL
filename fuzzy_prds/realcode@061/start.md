## Product Requirement Document

Hey team, we need to wrap up the MongoDB adapter library that lets devs skip all the usual boilerplate when talking to Mongo. You know how it is — everyone ends up writing the same callback-to-promise conversion code over and over, and it's a mess. The idea is basically the same pattern we used in that older driver shim project (you can dig around the repo to find how we handled the connection shorthand stuff back then).

The library needs to handle the full lifecycle: connecting with short-form addresses, grabbing collections by name (even names that clash with built-in JS properties), running queries, inserts, updates, deletions, and the atomic modify-and-return thing. Cursors are a big deal here — they need to support the whole range of iteration styles including streaming and piping, and they should be rewindable. Bulk write support is also required, including the big-batch splitting behavior and the plan-inspection feature before you actually execute. Collection admin stuff (stats, indexes, renaming, dropping, capped detection) needs to work too.

One thing that kept coming up in user feedback is that when you do a removal or update, the count reporting needs to match exactly what the server says — people kept getting confused about whether 'matched' vs 'modified' counts were right. Same deal with the find-and-modify — returning old vs new doc depending on a flag. Make sure error handling for duplicate users throws the right domain error type with the correct code. Details on exact field shapes for bulk plan inspection are in the test files.
