## Product Requirement Document

Hey team, we need to build out the coverage tracking system we've been talking about. Basically the idea is that we can record which lines of code actually ran during a real session, save that somewhere, and then pull it back later. We want it to work with at least two different storage options (like the one we used in the caching layer before, plus a simple file-based one). The tricky part is that if we run the same app on multiple servers or across deploys, the file paths are going to be totally different — we need some way to reconcile those so the reports make sense. Also we need merging: if the same file ran twice across two sessions, we don't want to overwrite the old data, we want to add it up. There should be a filtering step too so we can exclude vendor stuff and generated files. Oh and similar to the exclusion logic we wired up in that other collector module, there are a couple of built-in files that should always be excluded automatically on top of whatever the user specifies. The whole thing should be cleanly split into separate files/modules, not one big blob. We'll need a test runner that goes through a folder of JSON test cases and checks outputs. Let me know if anything is unclear but I think the shape of it is pretty obvious from context.
