## Product Requirement Document

We need a reusable abstraction that wraps a single persisted value — identified by a string key stored in a backing key-value store — and exposes it as an observable stream. When you write a new value it should persist to the store and push the update to every active observer. When you remove the value it should be deleted from the store and every observer should fall back to the configured default. When a new observer is attached, it should immediately receive whatever value is currently in the store (or the default if nothing is stored), without any write happening. The stream should deduplicate: if you write the same value twice in a row, the observer only fires once for the first change, not again for the repeat. Multiple observers on the same value should all receive every change in attachment order. If the value is constructed with no key at all, both write and remove should be rejected with a clear error — handle that the same way the validation layer handles empty required fields: refuse the operation and report why.
