## Product Requirement Document

We need a toolkit that automatically generates type-safe bindings for calling dynamically typed Python functions from a statically typed host language. Developers currently have to hand-write interop wrappers for every Python function they want to call, which is tedious and error-prone. The tool should read Python source text, extract every function definition header (including multi-line headers and those with trailing comments), and produce a ready-to-use method signature in the host language for each function found. The function name should be converted to the host language's upper-camel naming style, each parameter name should follow the lower-camel convention, and reserved words in the target language must be automatically escaped with a prefix character so the generated code compiles without modification. Type annotations should be translated to the closest strongly-typed equivalent in the host language, with well-known collection types mapping to idiomatic generic interfaces, tuple types handled specially depending on how many elements they carry, and anything unrecognized falling back to a generic dynamic object type. Default values must be preserved in the generated signature using the host language's own literal syntax, including numeric bases for hex and binary literals. When a function header cannot be parsed, the tool must report the exact character position of the failure rather than silently skipping it. The error reporting approach should follow the same pattern as the identifier validation module. The toolkit must also be usable as a standalone command-line adapter that accepts a JSON command on standard input and writes plain text to standard output, with all error conditions reported as neutral category lines rather than raw runtime exceptions.
