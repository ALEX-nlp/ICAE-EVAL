## Product Requirement Document

Hey team, we need to build out the core scaffolding toolkit primitives — basically the plumbing layer that the rest of the generator tool sits on top of. Think of it like that path-handling util we did for the deploy pipeline last quarter, but more comprehensive. The main pain points right now are: (1) every part of the tool is hand-rolling its own file relocation logic and they all behave slightly differently when there are parent-directory hops involved, (2) nobody agrees on what counts as a 'local' vs 'remote' module source so the fetch logic keeps breaking for certain repo reference styles, (3) manifest files are being parsed ad-hoc in at least three places and the parameter ordering is inconsistent in the output, and (4) parameter values that need to come from a shell command at runtime are being handled inconsistently — sometimes the newlines get stripped, sometimes they don't. We want a single clean core with a CLI adapter on top that reads JSON from stdin and writes results to stdout. There should also be a test harness that can run case files from a directory and save raw output per case — the naming convention for output files should follow the same pattern we settled on for the integration test runner. The whole thing needs to be properly structured, not a single giant file. YAML parsing will be needed for the manifest stuff. Let me know if you have questions.
