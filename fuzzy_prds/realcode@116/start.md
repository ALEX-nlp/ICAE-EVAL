## Product Requirement Document

We need a small, reusable helper library that supports a dependency-version reporting tool used by our engineering teams. The library must handle several common low-level tasks that come up repeatedly when scanning projects for out-of-date third-party packages. First, it should be able to break apart a version identifier string into its individual numeric and label components in a consistent order, so downstream logic can compare versions piece by piece. Second, it must classify how significant a proposed upgrade is — ranging from no change at all, up through incremental fixes, feature additions, and full breaking releases, with a separate category when version information is missing. Third, it needs a safe map-lookup helper that returns a sensible zero-equivalent fallback instead of crashing when a key is not found. Fourth, we want a dedicated error type for invalid user input that is recognizable as a standard error object and can optionally preserve the original underlying cause. Fifth, the library should enumerate the exact allowed values for each of the tool's command-line option categories, in their declared order, so input validation and help text stay in sync with actual behavior. Sixth, it must provide a fixed color assignment for each semantic role in the report output — project identifiers, framework labels, package names, and each upgrade severity level — so the visual rendering is consistent everywhere. All behavior is exercised through a simple command-dispatch protocol over standard input and output.
