## Product Requirement Document

We need a shared in-memory name registry and address resolver for a host that runs many isolated virtual networks, each containing multiple workloads that must discover one another by friendly name rather than raw address. A configuration loader reads a simple text-based description for each virtual network — the network's gateway listen points and the membership list of workloads attached to it, including their addresses and the aliases they should be reachable by — and builds lookup tables once at startup. The library must then support six query types: report the listen addresses of all networks broken down by address family; translate a workload name into its addresses from the perspective of a requesting caller; translate a raw address back into the names that own it; retrieve the custom upstream resolvers declared for a specific workload; retrieve the shared upstream resolvers inherited by all workloads on a given network; and report which network or networks a given address belongs to. Name matching must be case-insensitive so that mixed-case registrations do not cause misses. When a name lookup finds no match, the system must signal a non-existent domain rather than returning empty success. A single malformed network description must be silently skipped — it must not abort processing of the remaining well-formed networks. The core domain logic must be cleanly separated from any command-dispatch or input-parsing layer, and the dispatch layer reads a structured operation request and routes it through the internal resolution engine.
