## Product Requirement Document

We're building the reporting layer for a dead-code scanner tool. The scanner already produces raw lists of problems — imports that couldn't be resolved, declared dependencies that are never actually used, and source files that nothing ever imports — and a separate list of what was cleaned up. We need a library that turns these raw results into consistent, human-readable terminal reports so every part of the tool renders results the same way. There are three things the reporting layer needs to do: first, a small helper that normalizes any value into a list (so code that accepts either a single item or a list can always work with a list uniformly); second, a scan results report that shows a summary header with the version and entry point information and then a boxed table for each non-empty problem category with numbered entries; and third, a cleanup report that shows which dependencies were removed and which files were deleted, using boxed tables for non-empty groups and a confirmation line when a group is empty. When the scan finds no problems the report is just a single confirmation line. All reports must fit within an 80-column terminal width and must not use any terminal color codes — handle the layout the same way the existing table-drawing utility does. The box-drawing characters and exact spacing must match the reference implementation exactly so outputs can be diffed.
