## Product Requirement Document

Hey team, we need to get the terminal log colorizer shipped for the browser-based job view. Basically right now when you look at build logs in the UI you either see a wall of garbage escape characters or everything is plain unformatted text, and users are complaining it's unreadable especially for error output from tools like lager or webpack that do heavy terminal styling. We need something that takes that raw console output and turns it into proper HTML so it looks right in the browser. There are a few tricky bits — some of our internal log framework embeds special metadata payloads inside concealed text regions (you know, similar to how we handled that marker injection thing in the annotation layer before), and those absolutely cannot get stripped out even though normal hidden text should be. Also we have a couple of named color themes that need to work differently — one of them should wrap the whole output in a container with default colors, and colors that aren't covered by a named theme should fall back to indexed or rgb values. The action marker insertion feature also needs to spit out a specific output format with a normalized placeholder for the action id. Ping me if anything is unclear but let's try to get the core rendering logic separated from the I/O stuff this time, we got burned last time by having everything tangled together.
