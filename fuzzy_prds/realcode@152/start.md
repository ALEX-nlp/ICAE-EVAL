## Product Requirement Document

Hey team, we need a Python library that handles URL/resource reference stuff in a structured way. Basically devs are constantly writing the same brittle string-splitting code to pull apart URLs and it keeps breaking in weird edge cases — especially around passwords leaking into logs, internationalized hostnames, and percent-encoded stuff not being handled consistently. We want something clean that parses a URL into its pieces, lets you navigate around (like going up a directory, resolving relative links), manipulate query params without losing ordering or those weird valueless flag params, and convert between the human-readable Unicode form and the ASCII-safe form. There should also be a way to register custom schemes with their own default ports and authority rules, similar to how we handled custom protocol metadata in that config-registry module from the auth service refactor.

Also important: when something is broken/malformed, it should NOT bubble up a raw Python exception with internal stack details — it needs to output a friendly category name so our API consumers aren't seeing implementation noise. Same deal for the password redaction thing; make sure that's handled consistently across all the output paths. We also want structural equality comparison for URLs so caching layers can deduplicate properly. The output format for all of these operations needs to be consistent and machine-readable. Please align with however we did the encoded vs decoded representation split last time.
