## Product Requirement Document

Hey team, we need a structured logging toolkit built in Go. The core idea is that log events are just flat lists of alternating keys and values, and we want to be able to compose loggers that remember context, tag things by severity, swap out where logs go at runtime, and handle values that need to be freshly computed each time a log line is written. We also need to handle legacy log lines coming in from older parts of the system — kind of like that stdlib adapter pattern we used in the old observability work — and convert them into the same structured format. There should be two output formats: one human-readable and one machine-parseable. The whole thing needs to be split across multiple files, not just dumped into one giant file. There's also a runner/adapter layer that takes JSON commands over stdin and exercises the library, which is how the test harness talks to it. A few things I'm not 100% sure about: how exactly the timestamp parsing works for those legacy lines, what happens when the sink hasn't been set yet, how severity labels are named by default, and the exact ordering behavior when you combine context layers with the JSON format. Can someone clarify those details before we start? The error representation in the wire format is also a bit fuzzy to me.
