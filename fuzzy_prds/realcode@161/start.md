## Product Requirement Document

We need a two-factor authentication engine that a PHP application can use to add a time-based one-time password second factor. The core capability is taking a shared secret and a point in time and generating a short numeric code; the server and the user's authenticator app must agree on the same code at the same moment. The engine should support verifying a user-submitted code with a configurable tolerance window to absorb small clock differences between the user's device and the server — importantly, the tolerance window only looks backward in time, not forward. Once a code has been accepted, it must be remembered so a second attempt with the same code at the same moment is refused even though the code is still time-valid. The engine should also produce the standard provisioning URI that authenticator apps import by scanning a QR code, and it should be able to present the raw secret as well as a grouped version with spaces every four characters for manual entry. Finally, there should be a recovery-code system: generate a batch of single-use alphanumeric codes of a configurable length, and allow spending them one at a time (a code that is already spent or does not exist reports failure rather than throwing an error). All outputs must be predictable machine-readable strings, not localized messages or exception dumps.
