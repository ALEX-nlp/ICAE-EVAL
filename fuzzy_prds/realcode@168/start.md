## Product Requirement Document

Our media-session layer exchanges stream direction information between the configuration/API side and the low-level protocol side. Right now each component invents its own names and numeric codes for whether a stream sends, receives, both, or neither — leading to silent mismatches at boundaries. We need a small centralized utility that owns the canonical names and numeric codes for all recognized direction states, and provides two conversions: given a text name, return the canonical token for that direction (folding any unrecognized or mis-cased name into a well-defined 'undetermined' value rather than letting garbage through); and given a numeric code, return the readable token (mapping the well-known code range to their names and using a distinct catch-all for anything outside that range). The utility should handle validation errors for missing fields by emitting a structured error response. Make sure the 'undetermined' sentinel used for unknown names is distinct from the catch-all used for out-of-range numeric codes — look at how the peer-connection layer distinguishes these two error states and follow the same convention.
