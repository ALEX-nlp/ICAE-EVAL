## Product Requirement Document

We need a feature flag engine for our PHP application that lets us roll out new capabilities gradually and toggle them per-entity without touching the codebase for every change. The idea is to register each flag by name with a resolver function that computes its initial value — the resolver can return a constant, check the entity it is being evaluated for, or draw from a probability lottery. Once registered, a flag is considered 'on' unless its value is exactly boolean false; every other value (zero, empty string, null, complex objects) counts as active. Callers can override flags at runtime to force them on or off, and can apply that override to all currently known entity-scope combinations at once. Checking a flag without specifying an entity should automatically use whoever is currently logged in as the entity; and the empty-string entity should be treated as a distinct slot from 'no entity'. Each (flag, entity) pair should be resolved at most once and cached; there should be a way to pre-load values and to discard cached values to force fresh resolution. The engine should also support reading the concrete value of a flag (not just its on/off state), querying multiple flags at once, gating execution on whether a flag is active or inactive, and checking whether middleware should block an HTTP request when required flags are missing. Error cases — like requesting a concrete value for multiple entities simultaneously — should produce machine-readable category codes, consistent with how the auth and scope modules report problems.
