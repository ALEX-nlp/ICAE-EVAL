## Product Requirement Document

Hey team, we need a maps API wrapper library that handles building request URLs and parsing responses. The idea is developers shouldn't have to manually stitch together query strings or dig through nested JSON — the library takes structured inputs and spits out the right URLs or parsed summaries. We support a bunch of different service types: routing between two points, matrix distances, address lookups, place searches (nearby, text-based, details, photos, autocomplete variants), plus the timezone and geolocation ones which behave a bit differently from the rest (remember how we handled the alternate host situation in that geolocation spike last quarter — same idea here). For place photos specifically there's a size constraint thing we discussed — refer back to how we handled the 'at least one required field' pattern from the nearby search ranking validation. Coordinate pairs should always come out as lat,lng without any encoding. Address strings on the other hand do need proper percent-encoding. For the parsing side, when we get back raw JSON from any of these services we want a flat summary output — status, key counts, first meaningful fields — not a full object model. Also there's a shared time value thing for opening hours that needs to handle day indexing and reject malformed time strings with a neutral error. Calendar time format is exactly 4 digits or it should fail. Make sure the multi-origin/destination matrix endpoint joins entries correctly and that the transit mode list preserves order with pipe separators.
