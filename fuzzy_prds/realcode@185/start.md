## Product Requirement Document

Hey team, we need a library that auto-generates Markdown reference docs from our provider schema definitions. Right now devs are hand-writing all the field documentation and it keeps going out of sync whenever schema changes — it's become a real maintenance headache. The idea is: feed in a schema (with all the attribute/block metadata), get back clean, consistently formatted Markdown. We want the same grouping logic we used in that routing/classification work from the config module — required stuff first, then optional, then read-only. Nested structures should expand into their own sub-sections with anchor links. Each field needs a compact inline label showing the type and any special markers (like if something is sensitive or going away soon). There are also some smaller text utility helpers we need bundled in — stripping formatting from Markdown text, prefixing lines, and wrapping file contents in fenced code blocks (with graceful handling when the file doesn't exist). The whole thing should be modular, not one giant file. Types, labeling, classification, schema rendering, and text helpers should all live separately. The JSON wire format for types supports both bare strings and nested arrays — make sure the rendering handles deep nesting correctly. Can someone take a look at the test cases in the rcb_tests folder and make sure everything lines up with those contracts?
