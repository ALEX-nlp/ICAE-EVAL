## Product Requirement Document

Hey team, we need to build out our content moderation pattern engine. Basically the idea is that trust & safety folks can define 'bad phrases' with some lightweight pattern syntax (think optional segments, wildcards, word boundaries), and the engine handles finding those phrases in user text, then blotting them out. We also want to be smart about adversarial obfuscation — l33tspeak, lookalike Unicode chars, stuff like that — so normalization needs to be configurable in a pipeline style. There's also a whitelisting concept so something like 'pen is' doesn't trip on a partial match, similar to how we handled the safe-word carve-outs in that old classifier module. Repeated-character spamming (like 'heeeeello') should be collapsible too. On top of that, phrases should be groupable with metadata so downstream consumers know which category a match belongs to — and datasets should be mergeable/filterable. Output-side, we need a censoring layer that can mask matched spans in a few different ways (full mask, keep first char, keep last char, fixed replacement string). Error feedback should be clean and consistent — no raw stack traces or language-level exception names leaking out to callers. The whole thing should be wired up so JSON commands drive execution from stdin, with results on stdout. Would love this done cleanly so adding new transforms or strategies later doesn't require rewriting half the engine.
