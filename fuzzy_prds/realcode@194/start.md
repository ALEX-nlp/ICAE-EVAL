## Product Requirement Document

We need a guided-tour engine that walks users through an unfamiliar interface one step at a time. The engine should maintain an ordered list of steps, track which step is currently showing, and let the developer navigate forward, backward, or jump to a specific step by its identifier. Steps that are not relevant in certain conditions should be automatically skipped during navigation — similar to how conditional visibility works in the existing modal module. The engine should fire lifecycle notifications so the rest of the application can react: when a step becomes visible, when the tour completes, and when it goes inactive. Advancing past the last step should automatically finish the tour. Jumping to a step identifier that doesn't exist should be a no-op with no side effects. The engine should also support adding new steps after the tour is built, looking up a step by its identifier, removing steps by identifier (and if the removed step was the one currently showing, automatically advance to the first remaining step). Underpinning all of this is a small event notification system where callbacks can be registered under named event keys, fired with an argument, removed individually or all at once, and optionally registered as one-shot (auto-removed after first firing). There is also a utility for normalizing a placement specification from a compact shorthand string into a structured object, and one for expanding space-separated option strings into keyed objects.
