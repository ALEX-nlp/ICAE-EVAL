## Product Requirement Document

Hey team, we've got a request from one of our enterprise partners who's been complaining that their mapping integration keeps breaking whenever they try to build requests for places with special characters in addresses, or when they're passing in lists of stops for routing. They also mentioned something about the authentication tokens not being attached consistently — apparently sometimes requests go out without credentials at all, which is a compliance nightmare for them. Can someone look at how we handle the serialization side of things? I think there's some existing logic in the repo similar to what we did for the login module's encoding pass — just follow that same pattern. Also, we need to make sure that when the backend returns something other than a 200, we're not leaking internal exception details to the client — just a clean category label and whatever status number came back. One more thing: the partner also wants the roads and elevation endpoints to properly reject bad input shapes instead of silently doing the wrong thing. I know we've talked about this before but let's make sure it's actually enforced. Please check the existing test structure before adding anything new — I don't want duplicate coverage.
