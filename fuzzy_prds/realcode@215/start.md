## Product Requirement Document

Hey team, we need to wrap up the static repo generator tool we've been building. The core stuff is mostly spec'd out but there are a bunch of edge cases and behavioral details that got left fuzzy in the original notes. Basically the tool needs to handle six main workflows: picking packages from repos, scrubbing unsafe download URLs, generating the metadata files clients consume, loading back previously-generated metadata for incremental builds, rendering the browsable HTML catalog pages, and figuring out archive storage/skip logic. A lot of the tricky bits are around things like — when do we actually drop a package entirely vs just null out one of its URLs? How does the hash in the include filename get computed? What counts as a 'dev' package when we're loading existing metadata back? The abandonment reporting also needs to distinguish between 'replaced by something else' and 'just abandoned', which the frontend folks care about for the notice UI. Also there's that pattern we used in the login module for handling the host-matching rules — same idea applies here for the strip-hosts config, just refer back to how we did the CIDR + local path matching there. Output format for every command needs to be exact since it feeds into downstream tooling. Would love to get this fully nailed down before the sprint ends.
