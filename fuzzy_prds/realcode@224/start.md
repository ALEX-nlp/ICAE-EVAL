## Product Requirement Document

Hey team, we need to build out a bitmask flag toolkit that our platform teams keep reinventing every quarter. The core idea is simple: developers declare a named set of on/off options backed by an integer, and the library handles all the combining, querying, and printing so nobody has to hand-roll bit math anymore. We've had a few incidents where unknown bits sneaking through caused silent data corruption downstream, so there needs to be a clear story around how 'stray' bits are handled — some callers want them rejected, some want them silently stripped, some want to keep them. Also refer to how we handled the 'external catch-all' pattern in the options registry module, that same concept applies here for sets that need to cover every possible bit. The toolkit should support the usual set algebra people expect (combine, subtract, overlap checks), plus the ability to iterate which named flags are active in a value, serialize/deserialize to a human-readable pipe-separated format, and a classify/dispatch helper for routing values through labeled buckets. There's also a non-ASCII flag name situation that came up from the i18n team — make sure that works. Output formatting for the adapter layer needs to be consistent across all operations. Let's get this scoped out properly.
