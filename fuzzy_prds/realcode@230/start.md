## Product Requirement Document

Hey team, we need to build out that autoscaler config toolkit we've been talking about. Basically the idea is that ops people shouldn't have to write all this boilerplate parsing and merging code every time they stand up a new environment. The system needs to handle loading settings from files or directories (similar to how we handled the multi-file approach in the deploy pipeline thing we did before), merging layered configs where later values win, and translating those scheduler policy documents into something the scaling engine can actually consume. There's also some canonicalization step where we fill in sensible defaults — I think the built-in metric source and target names matter here but I don't remember the exact names off the top of my head, check the existing code. One tricky part: when the built-in metric source is active and the query looks like a short form (no slashes in it), we need to expand it somehow using group and job info. Validation also needs to happen and report multiple errors at once rather than stopping at the first one. The output needs to go to stdout in a consistent line-by-line format. Editor temp files should be ignored when scanning directories. Please make sure the architecture is clean — separate concerns properly, don't dump everything in one file.
