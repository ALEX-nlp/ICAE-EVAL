## Product Requirement Document

We need a template engine that lets developers write HTML as a terse, indentation-based outline instead of hand-writing verbose angle-bracket tags. Each line in the source represents one element; nesting is expressed purely by indentation depth so closing tags are automatically inferred. The language should support compact shortcuts for attaching IDs and class names to elements, a flexible attribute model that sorts attributes alphabetically and handles boolean values cleanly (true renders as an empty attribute value, false omits the attribute entirely), inline and multi-line verbatim text blocks, dynamic expressions that HTML-escape by default with an opt-out for raw output, and string interpolation inside text and attribute values. The engine should support control-flow constructs like conditionals and loops, code comments that are stripped entirely, HTML comments, and doctype declarations that map short identifiers to the correct full declaration string. There should also be a pretty-printing mode that indents the output for readability. When the source is malformed, the engine must reject it before producing any output and report the exact line and column of the problem. Some constructs that the grammar accepts are still forbidden by the engine's rules — handle those consistently with how the parser reports its own errors. Attributes from a hash or key-value collection can be injected onto an element via a splat operator, and repeated class values from multiple sources should merge into a single space-separated list.
