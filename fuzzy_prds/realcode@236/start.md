## Product Requirement Document

We need a small utility toolkit for our desktop application that solves two recurring pain points. First, our app needs to remember and restore window positions between sessions. We store everything as plain text in a simple settings file, so we need a reliable way to convert a screen region — defined by its origin corner and its size dimensions — into a text snapshot and back. Numbers must always appear in their decimal form even when they are whole, so the format is consistent. Second, our app maintains live snapshots of running programs and their on-screen windows. These snapshots must be treated as read-only values — you never mutate one; instead you produce a modified copy where only the fields you care about change and everything else stays the same. Two snapshots should be considered identical whenever their contents match, regardless of whether they are the same object in memory. The process snapshot holds an identifier name, a numeric handle, and a lifecycle state chosen from three possible states (one normal, one paused, one indeterminate). The window snapshot wraps a process snapshot and adds a numeric window handle and a display label. For the lifecycle state field, please look up what values the existing domain model already recognises so nothing is invented from scratch. The copy-with-overrides pattern in this codebase follows a specific convention — check the relevant model file to understand exactly how unset optional parameters fall back to the original value.
