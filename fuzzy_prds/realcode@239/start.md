## Product Requirement Document

Hey team, we need to ship the marketplace service client toolkit we've been talking about. The core idea is that devs shouldn't have to hand-roll all the low-level protocol stuff every time they build an integration — things like figuring out which endpoint to hit for a given region, expanding those weird indexed parameter lists the API needs, making sure request bodies have the right integrity checks, and parsing the XML blobs that come back. We also need it to build ready-to-send parameter sets for the main API areas — catalog/product stuff, orders, reports and feeds, inventory/finances, and the inbound fulfillment flows (remember how we handled the address normalization on that old shipment module? same vibe). The architecture should be clean — no giant single files — and the JSON I/O layer should be totally separate from the actual business logic. Credentials should come through as environment-like config (access key, seller account, auth token) and should show up in every outbound parameter map automatically. Timestamps in params need to be URL-encoded ISO-8601. Errors should use neutral category labels, not raw exception names. We'll be running a black-box test harness against stdin/stdout so make sure the adapter layer is solid. Let me know if anything's unclear on the region coverage or which seller ID field name to use for reports vs other areas.
