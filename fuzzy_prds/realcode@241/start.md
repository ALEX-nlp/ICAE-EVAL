## Product Requirement Document

We need a lightweight assertion and expectation engine for our test framework. The idea is that every check you write is both evaluated (does it pass or fail?) and rendered (what text describes the check?). The rendering is generated from the same expression that's evaluated, so you can never have a message that disagrees with what was actually checked. Six kinds of checks are needed: simple comparisons between two values (equal, not-equal, less-than, greater-than, etc.); logical compositions combining two comparisons with AND/OR or negating one; approximate equality with a tolerance margin; type-identity comparisons checking if two type names are the same; exception behavior matchers checking if code throws or completes normally; and named tests that group a set of checks together and report aggregate counts. The rendering format is strict: comparisons use mathematical symbols, logical compositions wrap the sub-expressions in parentheses, negation uses the word 'not' without extra parens, approximate equality shows the tolerance inline as '+/-', and type names appear verbatim. For approximate equality, pass means the difference is STRICTLY less than epsilon — same strict comparison rule as our numerical-tolerance module uses. Named tests report whether they ran or were skipped. Skipped tests never evaluate their body.
