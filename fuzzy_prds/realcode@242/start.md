## Product Requirement Document

We need a management layer in PHP that drives the system package manager to install, link, and control packages and their background services, so our local development tool can manage its own runtime dependencies through a clean interface without scattering raw shell strings everywhere. The layer should be able to check whether a named package is installed (by inspecting the metadata the package manager returns), determine whether any supported PHP version is currently installed from a fixed list of recognized version names (both the dotted style and the older compact style), register external package repositories before installing from them, start and stop background services (service restart requires a plain stop then a privileged stop then a privileged start; service stop requires plain stop then privileged stop), install a package with optional repositories and fail loudly with a normalized error if the manager exits non-zero, link and unlink package binaries with similar error handling, and detect which PHP version the package manager has linked by inspecting the symlink on disk. All shell commands issued must be traced in order in the output as 'cmd=...' lines so callers can verify the exact commands used. The invoking user name for user-context commands is a fixed value in the test environment. Errors must be machine-readable category codes, the same approach used by the PHP detection and service-control parts of the system.
