## Product Requirement Document

We need a reusable polymorphic container library for C++ that allows any object meeting a declared set of behavioral contracts to be stored and operated on uniformly, without requiring a shared base class or inheritance hierarchy. The library must support four main scenarios: (1) A container that holds any of several unrelated shape-like objects and can invoke both a text-rendering operation and a numeric measurement — results use fixed-point notation with five decimal places, and unrecognized or mis-parameterized object kinds are rejected with a structured error signal; (2) A single named operation that carries multiple overloads dispatched by argument count, so callers with different argument shapes reach the correct implementation automatically; (3) An operation declared as optional, allowing the container to hold collections that may or may not implement it — when unavailable, a well-defined not-supported signal is raised instead of crashing; (4) Run-time type inspection and value recovery: the container reports a tag for what it holds, and a stored value can be retrieved by naming its expected type — a mismatch produces a neutral type-mismatch error. A thin adapter program reads a JSON command from standard input, routes it by action name to the appropriate handler, and writes results to standard output. The automated test runner script under rcb_tests saves raw per-case output for comparison. The core type-erasure machinery — concentrated in the main header file of the library — must remain completely decoupled from any I/O concerns.
