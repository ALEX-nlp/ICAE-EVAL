## Product Requirement Document

We need a small utility library that helps our tooling work reliably with hosted code repositories and download links. The core problem is that the same repository can be referenced in multiple equivalent ways — some use a shorthand connection format while others use a standard web address format with optional embedded credentials — and we need a single consistent identifier regardless of which form was used. Additionally, download links often include volatile session tokens and analytics markers that change on every request, which makes it impossible to compare or cache links reliably; we need those stripped out while preserving any parameters that are genuinely part of the resource address. The library should expose two clean, focused functions: one that converts any recognized remote address format into a stable owner-slash-repository string, and one that returns a sanitized version of a URL with ephemeral tracking noise removed. Handle edge cases gracefully — follow the same error contract we use in the authentication module so callers get a meaningful exception rather than a silent failure. The output format for the identifier function should mirror how our existing pipeline identifies repository slugs. We also need a thin command-line adapter that accepts a JSON request on standard input and routes to the appropriate function, printing the result to standard output, so automated test harnesses can drive the library without importing Python code directly.
