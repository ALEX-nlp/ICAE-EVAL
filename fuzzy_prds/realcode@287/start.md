## Product Requirement Document

Hey team, we need to ship the interweave rich text renderer. Basically it needs to handle a bunch of stuff — taking user content (could be plain text, could be HTML snippets) and spitting out safe rendered HTML. Core things: wrap content in the right container tag (or just a fragment if needed), handle when people paste in HTML with dangerous stuff like scripts or onclick handlers (strip those out), normalize attributes so things like datetime and aria- and data- attributes survive but sketchy ones don't, and auto-convert line breaks in plain text to break tags (but not when disabled). Also needs to linkify content — emails, hashtags, regular URLs, and IP-based URLs should all become clickable anchors, but don't nest anchors inside existing anchors. There's also emoji stuff — matching unicode, shortcodes, emoticons and packaging emoji records with canonical shortcode/primary shortcode/skin metadata. For validation, non-string content and full HTML documents should come back with the right neutral error codes — similar to how we handled the rejection logic in that auth-input module we did before. Pattern matching for emails, hashtags, URLs, and IPs needs to expose the individual parts too. Make sure the core logic is totally decoupled from the stdin/stdout adapter layer. Refer to the existing test cases for exact output shapes.
