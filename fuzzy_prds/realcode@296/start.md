## Product Requirement Document

Hey team, we need a serialization utility that can look at our data types and automatically figure out what the wire format should look like — basically no more hand-writing schemas every time someone adds a field. It should spit out something machine-readable (JSON-based) that describes the shape of the data, and also be able to turn actual values into a compact byte sequence and read them back. Think of it like what we did with the nullable handling in the profile service last quarter — same kind of 'wrap it in a union' pattern.

The tricky bits: integers have some collapsing behavior (the narrower ones fold into a single wire type), characters are special-cased with a marker, and text needs a length prefix baked in. For enums specifically, we want the schema to carry documentation, aliases, and a designated fallback symbol — and if someone misconfigures the fallback (more than one), it should fail loudly with a recognizable error token, not a stack trace. We also need the decode side to be tolerant of schema drift, so if the reader's enum is a subset of the writer's, unknown symbols resolve gracefully. Output bytes should be lowercase hex. Please keep the I/O wiring totally separate from the core logic — last time everything got tangled and it was a nightmare to test.
