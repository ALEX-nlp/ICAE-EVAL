## Product Requirement Document

Hey team, we need to get the workflow manager CLI tool properly handling a bunch of the core interactions that keep coming up as pain points. Users are complaining that when they pick an agent from the list it sometimes does weird things with their input — spaces, out of range numbers, that kind of stuff. Also the color theming is broken for some people who paste in hex codes from their design tools. The first-launch experience is inconsistent depending on whether someone is a fresh install vs migrated vs returning user — we keep getting support tickets about this. Config merging between global and project level isn't respecting the right priority order either, which is causing teams to wonder why their project-level settings get ignored sometimes. The board navigation feels off when columns have different numbers of tasks — refer to how we handled the clamping logic in the selection module from the earlier sprint. Workspace paths need to follow the hidden directory convention we settled on (check the existing path resolution stuff). The git worktree create/remove cycle needs to actually work against a real repo, not just simulate it. Skill mapping per agent family needs to follow the command convention each CLI expects — claude and opencode behave differently here. Also the terminal capture trimming is leaving too much blank space. Please make sure all the edge cases around blank input, invalid text, missing optional fields, and empty columns are covered properly.
