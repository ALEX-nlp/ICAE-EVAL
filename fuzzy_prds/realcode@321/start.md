## Product Requirement Document

We need a client library that wraps a remote infrastructure management service's REST interface. The library should allow operators to inspect and manage the host daemon — retrieving a summary of the daemon's environment (including its trust status, underlying storage backend, runtime versions, and kernel information) as well as reading individual fields from that environment. When the daemon is unavailable, field lookups should return a null value rather than raising an error. Network management should follow the same general pattern: list all networks by their short names, show details for a single network, or read individual fields. Async operations tracked by the daemon should be listable, inspectable, cancellable, and waitable. Containers form the core resource: operators should be able to list them, check whether they exist or are actively running, start/stop them with a force flag, create/destroy them, read state and logs, migrate them to other hosts, publish them as catalog entries, manage their snapshots, push or pull files, and execute commands inside them. The catalog of images should support listing, searching, existence checks, detailed summaries with human-readable sizes and dates, per-field access, export, upload, and deletion. The system must also manage named aliases for catalog entries, trusted client certificates, and configuration profiles — all following the same list/show/create/update/delete pattern as the network module. Errors from the backend should be surfaced as typed exceptions, and boolean helpers should indicate success or existence cleanly.
