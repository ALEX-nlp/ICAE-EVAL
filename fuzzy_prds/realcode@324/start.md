## Product Requirement Document

hey team, we need to build out that logging engine thing we talked about last sprint. basically the idea is devs should be able to fire off a log entry with some context attached and have it show up in one or more places without them having to do a bunch of wiring themselves. there's a bunch of stuff we need to handle — like, sometimes the message bytes are garbage and we don't want the whole thing to blow up, so we need to quietly fix that before it hits storage. also we talked about the thread thing, like how last time on the jobs service one noisy thread was drowning everything out — we need to be able to turn down the noise on just that thread without touching the rest. we also want the same entry to go to multiple destinations at once, kind of like how we set up the audit trail fan-out in the payments module. oh and there's a mode where you want the entry to go downstream but NOT get written to the main store — that needs to still fan out properly. entries should be able to carry extra context (stack info, environment stuff) without us manually passing it everywhere. the adapter layer should read commands from stdin as JSON and write results to stdout, same pattern as the notification dispatcher we built. deferred message producing should also be supported. check the existing test cases for the exact output format, i don't have time to spell it all out here.
