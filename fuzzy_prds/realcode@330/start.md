## Product Requirement Document

We need to finish the black-box surface for our transparent build accelerator, not just the wire serialization layer. The tool should expose a deterministic adapter for the same scenarios our tests cover: command-line/help and invalid option handling, direct command execution with stdout/stderr/exit status passthrough, descendant process interception across common execution methods, environment-fixup behavior for missing or hostile environment variables, diagnostic stream passthrough for warnings/errors, cache-hit acceleration behavior across repeated builds, and the typed wire-message serialization format used internally by the accelerator. Keep the core accelerator/process/serialization logic separate from the JSON adapter, and preserve the exact stdout contracts in the public cases.
