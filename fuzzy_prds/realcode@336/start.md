## Product Requirement Document

We need a set of file-inspection and code-understanding helpers for our coding assistant. Right now different parts of the tool re-implement language detection, query parsing, and file editing in slightly incompatible ways. We want one shared toolkit with a clean contract. Six operations are needed: first, map a file's extension to a programming language name — the assistant needs this to decide how to handle each file without reading its contents; second, extract likely code identifiers from a natural-language question — only meaningful tokens, filtering out short words and common programming stop-words; third, turn a question into a list of file glob patterns worth searching — language-family patterns based on trigger words found in the question, plus some sensible defaults; fourth, produce a quick structural outline of a source file using a line-by-line scan — function names, class names, structs, etc., up to thirty items; fifth, render a file's content with line numbers, with optional windowed slicing by offset and limit; sixth, do a safe in-place text replacement that only proceeds when the target string appears exactly once. The unique-edit feature should handle ambiguity the same way our config-file patcher handles it — if the target appears multiple times or not at all, report a specific error with the count, never silently corrupt the content. Line number rendering uses a four-character field, right-justified.
