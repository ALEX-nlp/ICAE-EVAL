## Product Requirement Document

Hey team, we need to ship the TypeScript/JS code formatter plugin. Quick summary of what we need: it should take messy source code and spit out clean, consistently styled output. Users have been complaining that their code looks different across machines and PRs are full of noisy whitespace diffs — super frustrating for the team. The formatter needs to handle all the usual stuff: spacing, indentation, string quotes, semicolons, trailing commas, that kind of thing. Should also support JSX and the CJS module variant (check how we handled dialect detection in the file-type logic we already spec'd out). One thing that came up in the last retro — people want to be able to drop a special comment to opt out of formatting for a specific node or the whole file, similar to what we did for the ignore patterns in that other plugin. Also needs to be robust: bad input or weird config values shouldn't crash anything, they should come back as a clean error message. The whole thing needs to be split into proper modules — no giant single-file dumps please, the last time someone did that it was a nightmare to maintain. Oh and make sure running the formatter twice doesn't change anything the second time around. Ping me if anything's unclear but I think the test cases cover most of the edge cases we care about.
