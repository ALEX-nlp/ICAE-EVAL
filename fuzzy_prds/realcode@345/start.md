## Product Requirement Document

hey team, we need to wrap up the zonal extraction tool we've been building. the core engine is mostly done but we're getting complaints from a few data science folks that the output tables are inconsistent depending on how they pass in polygons and rasters. basically the ask is: given a set of polygon features and one or more raster grids, we should be able to spit out a clean CSV (or shapefile in some cases) with whatever summary numbers they need per polygon. things like averages, most common value, spread of categories, that kind of stuff. there's also a bunch of edge cases around empty intersections and missing data that keep biting people — same kind of handling we did for the nodata sentinel logic in the reprojection module, just make sure that's consistent here too. one tricky bit: when someone only has a single unnamed raster, the column headers in the output should just use the operation name directly instead of inventing some raster prefix. also cell-level outputs need to work where each intersecting cell gets its own row. and there's some metadata about scaling/offsets on raster bands that needs to be applied before values go out. the identifier column in output should support renaming and type-casting. geometry passthrough to shapefiles should be supported when requested. please make sure fractional overlap is exact and not just a nearest-cell approximation — that was a big complaint last time.
