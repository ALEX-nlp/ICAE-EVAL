## Product Requirement Document

Hey team, we need to get this graph library wrapped up and working end-to-end. The core idea is that developers should be able to throw any kind of graph at it — the undirected kind, directed, the ones keyed by actual values, the stable-index ones where holes are okay, and also that union-find thing — and run all the usual algorithms without writing their own adjacency bookkeeping. Think shortest paths, traversals, cycle checks, spanning trees, component grouping, and even that structural equivalence check we talked about (similar to what we did for the equality logic in the mapping module, you know the one). The tricky bits are around mutation: when you remove a node everything touching it has to disappear cleanly, but unrelated stuff must survive untouched. Also the edge-update behavior needs to be consistent — no surprise duplicates when you update the same connection twice. The stable-index graphs are a bit special because outside code might be holding onto old IDs so we can't just renumber things after a removal. Error cases need to surface clearly rather than silently doing the wrong thing, especially when someone references an index that no longer exists. Output for the DOT format export also needs to be exactly right. We're using the petgraph crate as the backbone so make sure the adapter layer lines up with how that API actually works. Can someone own the full implementation and make sure all the public test cases pass?
