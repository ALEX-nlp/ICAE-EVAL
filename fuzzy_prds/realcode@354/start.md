## Product Requirement Document

Hey team, we need to wrap up the tree ensemble deployment toolkit. The core idea is that data scientists train models externally (XGBoost-style gradient boosted forests, including ones with those category-based branching conditions) and then hand them off to us — we need to load them, let engineers poke at the structure a bit, compile them down to something fast, and serve predictions. We've been burning too much time on every new model deployment because there's no standard interface, people keep shipping the full training stack into prod which is a nightmare, and scoring code keeps drifting out of sync with the actual model.

The interface should accept a JSON command on stdin and print key=value lines back — one action per invocation. We need metadata inspection, the ability to chop off trees from the back (like for ablation studies), compiled predictor info, batch scoring, single-row scoring with a few different ways to express missing values (refer to how we handled the null encoding in the feature store pipeline last time), and handling for when input data is the wrong width. Also need a serialization round-trip so models can be stored and reloaded without losing numeric fidelity. Oh and there are some compiler tuning knobs — quantization and that folding thing — those should never change the actual numbers output. Numeric outputs should be formatted consistently. Errors should be clean categories, not stack traces.
