## Product Requirement Document

We need a stream pipeline engine that lets developers compose a finite sequence of fallible values through declarative transformation stages and collect the result. Each element in a source sequence is either a successful integer item or an error carrying an integer code. Stages can map values, filter them by predicate, combine filter and map in one step, take or skip leading elements (counting only successful items, not errors), stop or skip while a predicate holds, rewrite error codes, chain fallible transforms on success or on any outcome, recover from errors by substituting a value, and combine two sequences by pairing their elements or by racing to whichever produces first. There is also a batching stage that groups items into fixed-size chunks. The terminal collectors either gather all items into a list, fold items into a single accumulator, or simply drain the sequence. A separate flatten operation concatenates groups together. Handle errors the way the standard transform chain handles them — when an error arrives it stops the collecting sink and that error code is reported, unless a recovery stage intercepts it.
