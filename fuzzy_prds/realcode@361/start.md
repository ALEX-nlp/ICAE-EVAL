## Product Requirement Document

We need a small library that lets numerical code describe the shape of a multi-dimensional array and then translate any multi-index into a flat memory offset, so engineers never have to hand-derive stride formulas again. The library should support three storage conventions: the natural row-major convention where the last dimension changes fastest (used by C arrays), the column-major convention where the first dimension changes fastest (used by Fortran), and a custom-stride mode where the caller supplies the step size for each dimension directly (useful for non-contiguous views, padded buffers, or broadcast layouts). On top of that, there should be a shape descriptor type that can express a mix of dimensions fixed at compile time and dimensions that are only known at runtime; the descriptor should be queryable for its total rank, how many of its dimensions are runtime-sized, the compile-time pattern with runtime slots shown as a placeholder, and the fully resolved sizes after the runtime values are substituted in. For all three mapping types the library should report the per-dimension strides, the minimum number of storage slots the mapping spans, whether the mapping covers storage without any gaps, and the single linear offset for a given multi-index. The custom-stride mapping should be flagged as non-contiguous regardless of the actual numeric values supplied, because contiguity cannot be assumed for caller-provided strides. Zero strides are allowed in the custom mode — a mapping with all-zero strides collapses every multi-index onto offset zero and spans only a single element.
