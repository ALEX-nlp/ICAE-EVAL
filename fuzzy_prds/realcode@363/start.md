## Product Requirement Document

Hey team, we need to build out the random forest engine we've been talking about. The core idea is that a developer should be able to point it at a dataset, say which column is the outcome they care about, and just get a trained model back without doing a bunch of manual setup work. We want to support the usual stuff — classifying categories, predicting numbers, and also that survival analysis thing we discussed (like what we did in the clinical data project last quarter, similar Surv() syntax). There should be a way to describe the problem either with a formula string or by just naming the column directly, both should work the same way. Data can come in as a typed table or as the all-numbers version of the same data, and the engine shouldn't care which one it gets. We also need proper error handling — not just crashes, but meaningful category labels for things like bad sampling settings, wrong split rules for the task type, missing values showing up, that kind of thing. Reproducibility is important so make sure seeding works correctly. We want OOB error, variable importance when asked for, confusion matrix for classification, probability outputs, per-tree prediction shapes, and survival timepoint tracking. Oh and warnings when something like unused categories get dropped during training. The adapter layer should be thin and separate from the actual engine logic — keep I/O concerns out of the core.
