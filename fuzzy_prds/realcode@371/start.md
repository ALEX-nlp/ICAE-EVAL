## Product Requirement Document

We need a small toolkit of composable primitives for a secrets-management command-line tool. The primitives cover six distinct concerns. First, parse an octal file-permission string as typed on a command line into a concrete permission — accept a leading zero or not, reject strings shorter than three characters, and report both the canonical four-digit octal form and the symbolic rwx representation. Second, ensure output text always ends with exactly one trailing newline so it plays nicely with shell pipelines. Third, validate environment-variable names against two rulesets: a portable-charset ruleset (which bans null and equals sign) and a stricter POSIX shell ruleset (letters, digits, underscores only, no leading digit). Fourth, format a calendar instant as a machine-readable timestamp given the date-time components and a named timezone — zones with no UTC offset should show 'Z', positive-offset zones should show '+HH:MM'. Fifth, expand dollar-brace placeholders inside a template string by substituting supplied values — trim spaces from key lookups, fail clearly if a key is missing or if an opening delimiter is never closed. Sixth, redact secret values from a stream of output as it is written, even when a secret spans multiple writes — handle it the same way the masker module handles overlapping secrets in other command outputs: longer secrets take precedence over shorter ones that share a prefix.
