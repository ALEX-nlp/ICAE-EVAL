## Product Requirement Document

We need a renderer that turns a thread's captured call-stack snapshot into a readable traceback, the way a debugger or crash reporter would show it. The renderer handles two views: a pure Python listing and a merged native/Python listing that interleaves C and Python frames. For the pure Python mode: print each frame as 'File ..., line N, in <function>', with the relevant source line beneath it when available. Internal placeholder frames should be skipped in this mode. Arguments and locals print below the source line with a specific header format. The thread header summarizes runtime conditions in brackets — GIL state and garbage collection status. Each traceback ends with a blank line. For merged native mode: we need to reconcile the native C frames with the Python frames by matching interpreter evaluation frames to Python call entry points. If they don't match, fall back to the Python-only listing with an unmergeable notice. When they do match, weave C and Python frames together. Interpreter-internal plumbing frames get filtered out — same filtering approach as our profiler uses for noise reduction. Inlined calls (multiple Python frames under one evaluation frame) all render together. In merged mode, those placeholder frames that were skipped in pure mode DO show up. The source line display depends on line numbers: a line number of 0 means show the last line of the available source.
