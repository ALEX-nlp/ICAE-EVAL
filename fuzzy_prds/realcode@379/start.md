## Product Requirement Document

We need a text rendering engine that accepts wiki-style markup and produces the fully resolved output string. The engine takes a piece of source text, an optional page title (which defaults to a standard placeholder name), and an optional list of supporting pages, each with a title and a body. Pages whose titles use a designated template namespace prefix can be transcluded by name inside the source text. The engine must substitute positional and named arguments in template bodies, support nested template calls, and handle special punctuation injection helpers. Conditional branches must evaluate based on emptiness, equality, multi-way matching, and numeric expression results. Inline error detection in already-expanded sub-values must be supported. Arithmetic expressions including trigonometric functions, constants, scientific notation, comparison operators, and integer division must be computed. String operations including substring, search, pad, replace, and split are required, along with letter-case conversion and number formatting with grouping separators. URL percent-encoding and decoding, page title path slicing, and namespace identifier resolution must be implemented. Page-title introspection and date formatting with format codes are also required. Error conditions must surface as inline HTML-style markers in the output, never as runtime exceptions. The input-output adapter layer that reads requests and writes results must follow the same separation-of-concerns pattern used in the authentication module, remaining strictly decoupled from the core engine logic.
