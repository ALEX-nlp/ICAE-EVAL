## Product Requirement Document

hey team, we need to ship the binary header codec thing for the encrypted message format. basically we need something that can take a header description and turn it into bytes and back, similar to what we did for the session serialization layer a while back (check how we handled the partial-read logic there, it should apply here too). the main pain points we've been hearing from devs are: 1) they keep screwing up the byte math when fields arrive in chunks over a socket and the whole thing crashes or silently corrupts, 2) nobody agrees on what an 'empty' or 'unbuilt' header should look like before it's been populated, 3) we've had a couple incidents where someone sent a header with no integrity protection attached and downstream just accepted it — that needs to be a hard stop. also there were some complaints about oversized metadata blobs blowing up in weird ways instead of giving a clean error. the codec needs to be split into sensible pieces, not one giant file, and the JSON/stdin stuff should be totally separate from the actual header logic. i'd say look at how the login module handled the compatibility validation last time for reference on the error reporting style. we need solid round-trip guarantees and the streaming/incremental decode is a must-have for the network team.
