## Product Requirement Document

We need a small collection of reusable utility helpers that our desktop application can use across many different screens and components. These helpers should cover a few everyday needs: cleaning up version label strings so they all look consistent, producing grammatically correct text when showing counts of things, turning a flat list of release records into a fast lookup table, ordering those releases so the newest ones always appear first (while still handling oddly-formatted labels gracefully), navigating and updating deeply nested settings objects using a simple path notation, building a visual split-panel arrangement from a list of panels and also reading that arrangement back into an ordered list, computing where a floating tip or detail card should appear on screen without going off-screen edges, and reordering action buttons in dialogs to match what the operating system expects. The version-cleaning logic should work the same way it already does in our existing string normalization module, and the button ordering behavior should mirror the platform-aware sorting we use elsewhere in our dialog management code. All helpers must be pure and dependency-light so they can be tested independently and composed freely. Outputs should be plain text or compact serialized data with no extra whitespace, and each result should end with a newline.
