## Product Requirement Document

Hey team, we need to build out a holiday/business-day calendar library — think of it like that date-utility thing we prototyped for the payroll team a while back, but much more complete and multi-country. The main pain points we're hearing: scheduling systems are breaking because nobody knows which dates are actual working days vs. public holidays, especially around Christmas/New Year when weekends shift things around. We need it to handle a bunch of countries (US, UK, France, Italy, Japan, Canada, NZ, Austria, Australia at minimum), and it should also support regional stuff like Canadian provinces or NZ districts because right now people are getting the wrong dates for BC vs Saskatchewan. Also need the ability to work with totally custom holiday lists for clients who have their own calendars. The date-shifting logic when a holiday lands on a weekend needs to follow proper substitution rules — similar to how we handled the observed-date logic in that other project. Business-day addition and counting should work correctly too. One more thing: the easter calculation needs to be baked in since a lot of the European holidays depend on it. Architecture-wise, please keep concerns properly separated — I/O and business logic should not be tangled together. Make it clean and extensible so we can bolt on new countries later without rewriting the core.
