## Product Requirement Document

We need a small, reusable numeric engine that takes a list of closing prices and a look-back window size and produces a full trend or momentum curve — one smoothed value per input price, in order. The engine must support at least four classic signal types: a plain running average, a recency-weighted average that reacts faster to new prices, a linearly tapered average that puts the most emphasis on the latest price, and a bounded momentum oscillator that compares upward pressure to downward pressure on a zero-to-one-hundred scale. All output values must be formatted as fixed four-decimal strings using standard rounding; a mathematically indeterminate result should be represented by a special not-a-number marker rather than crashing. Before a full window of historical data exists, each indicator should degrade gracefully to a shorter window rather than returning nothing. The system must expose a clean command-line adapter that accepts a JSON request on standard input and writes the curve to standard output; if the requested signal type is not recognised, the adapter should emit a two-line neutral error message. The core calculation logic must be fully decoupled from the I/O layer. Agents implementing this should look for the existing indicator hierarchy in the tracker sub-package and the decimal wrapper used throughout the library for exact rounding semantics.
