## Product Requirement Document

We need a small collection of general-purpose helper routines that our teams keep reimplementing from scratch on every project. The pain points are: (1) handling list positions that count from the end — negative indices should work the same way they do in the "standard utilities" we already use elsewhere; (2) merging one list into another with optional start/end bounds; (3) running a risky operation and capturing the outcome as a two-slot result without scattering try/catch blocks; (4) deciding whether two decimal values are 'close enough' to treat as equal; and (5) copying selected properties from one object into a fresh object, with the option to rename them, and correctly handling zero and empty-string values (not just truthy ones). We also need an in-place way to add extra properties to an object or to every object in a list, and a safe way to ask whether a subject actually has a given property before touching it. Please treat undefined/absent properties consistently with how the auth module handles absent values — look at how that convention is established in the codebase and follow the same sentinel pattern throughout.
