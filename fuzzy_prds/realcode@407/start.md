## Product Requirement Document

We need a toolkit that helps manage which versions of common JavaScript runtimes and package managers a project uses. When a developer types something like 'node at version 3' or 'yarn latest', the tool should understand what they mean — whether they pinned an exact release, asked for the newest, or gave a range — and normalize it into a consistent internal form. There's a tricky difference from other ecosystems: writing '1.2.3' here means exactly that version, not 'compatible with'. The tool also needs to figure out the right download file names for each platform (Linux x64 uses tar.gz archives, for instance). When someone passes arguments in the wrong order — like putting the version first without the tool name — it should catch that and report it cleanly rather than crashing. We also need a small config file that records the chosen versions; handle it the same way as the session-management module does for reading and writing structured JSON. The file should store node runtime, bundled npm, and yarn versions, printing null when something isn't set. The whole thing should be arranged so the parsing logic is separate from the file I/O code.
