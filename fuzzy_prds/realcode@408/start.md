## Product Requirement Document

We need a TypeScript/JavaScript source model toolkit for our editor tooling. It has four main jobs: parsing source files into a structured tree, rendering model elements back to source code, building a project-wide declaration index, and normalizing file paths and URIs. For parsing: given source text and the script kind (TS, TSX, JS, JSX), the toolkit should return a structured model of the file's imports, exports, declarations, nested modules/namespaces, and referenced identifier names. The model should handle all import forms — bare side-effect imports, named imports (with optional aliases), namespace imports, and CommonJS-style require imports. Exports should cover re-export-all, named re-export, and assignment exports. Declarations include classes (with constructors, properties promoted from constructor parameters, methods, accessors), interfaces, enums, functions (with parameters and local variables), variables, type aliases, and default exports. Source character spans (start/end) are included. For generation: given a model element and formatting options, produce the source text. Handle this the same way our snippet generator handles multi-line wrapping when lines exceed the threshold. For the index: build a reverse map of which file exports which name, following re-exports. For path normalization: standardize file paths and URIs for cross-platform lookup.
