## Product Requirement Document

Hey team, we need to build out the core period/scale/parameter simulation engine we've been talking about. The gist is: users want to pass in compact time expressions and get back structured breakdowns — like splitting a year into months, or figuring out which legislative rate was in force on a given day. There's also a bunch of bracket math involved (think progressive tax bands), and we need to handle the 'net from gross' recovery case that tripped us up last time with the payroll module. Household grouping is another big one — we need to figure out parent/child roles, ordering, and which household index each person belongs to. Oh and the axis expansion thing from the old simulation grid prototype should be folded in too. A few things I'm fuzzy on: not sure exactly how the collapsing rules work for the period string format (when does a month-based period collapse to just a year string?), and I don't remember what the exact rounding mode was supposed to be for bracket base values — can someone double-check the spec? Output always goes to stdout as key=value lines. Errors should never expose internal exception names. Let's make sure the folder structure reflects the complexity here, not just one big file.
