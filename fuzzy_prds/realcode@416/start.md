## Product Requirement Document

Our team needs a self-contained utility library that helps application developers safely assemble and inspect the two main text-based query languages used by centralized identity services. The first concern is neutralizing special characters inside arbitrary user-supplied strings before those strings are embedded in queries or address paths — the escaping mode should be selectable so only the characters relevant to the target context are touched, and specific characters can optionally be passed through unchanged. The second concern is building search queries from a declarative list of conditions rather than hand-writing fragile text: conditions can require an exact match, a range, set membership, or substring presence, they can be combined with AND or OR logic, nested into explicit logical groups, or supplemented with verbatim pre-built fragments. Every built query must expose both a wire-safe form and a human-readable form. The third concern is working with hierarchical address strings that identify individual entries in the directory tree: the library must decompose such an address into its constituent parts and expose named scalar views of the first component and the parent path, validate whether a string is a well-formed address, compare two addresses to determine ancestor or child relationships, and support an ordered sequence of non-destructive edits such as reversing, trimming from either end, or attaching new components whose reserved characters are automatically neutralized. Finally, a parser must convert an existing query string back into a structured node tree — following the internal node representation used by the group and condition types — and reassemble it into canonical form; malformed input must surface as a dedicated error signal rather than a silent failure. The team has flagged the escaping-mode selection logic and the DN hierarchy comparison as the two areas most likely to cause subtle bugs.
