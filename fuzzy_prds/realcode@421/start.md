## Product Requirement Document

Hey team, we need to wire up the OGNL expression evaluation runner for the upcoming demo. Basically we want a single entry point that reads a JSON blob from stdin, figures out what fixture to load (we have a handful of pre-baked ones — simple, sample, the array ones, list, etc.), evaluates the expression against it, and prints the result in that normalized two-line format we settled on. If there's a set_to field we should do the before/after thing like we discussed in the retro. Also make sure variables get seeded into the eval context before we run. For error cases just print the category label, no stack traces or anything noisy. One thing I keep forgetting to mention: the output format for numbers needs to match whatever we agreed on for the int-vs-float distinction — I think it's the same rule we used in the last formatter we shipped, so just check how that handles trailing zeros and the dot. Also the map result ordering was a pain point in QA last sprint so double-check that. The fixtures themselves need to match the sample graph from the existing test expectations, specifically the nested map structure, the disabled flag, the array contents, etc. We want the whole thing runnable as a standalone Java program fed JSON on stdin.
