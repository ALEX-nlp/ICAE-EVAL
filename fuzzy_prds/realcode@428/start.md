## Product Requirement Document

Hey team, we need a numeric utility library built out — basically a toolkit for doing low-level integer and fractional math in a portable, predictable way. Think of it like that bit-manipulation helper we shipped for the embedded project last quarter, but way more comprehensive. The big ask is: developers shouldn't have to hand-roll their own code every time they need to do things like inspect bit patterns, shift bits around cyclically, work with fractions exactly (no floats mucking things up), or deal with fixed-precision decimal-ish values. There are also some edge cases around what happens when numbers go out of range — we want that behavior to be explicit and configurable rather than just silently wrong. The library needs to be structured cleanly so it's not just one giant file, but also don't go overboard if something is genuinely simple. It should be testable via a CLI adapter that reads from stdin and writes to stdout. The test harness should be runnable with a single bash command and output per-case result files in a specific directory structure so CI can compare them easily. For the fraction stuff specifically, make sure you look at how we handled the normalization vs. canonicalization distinction in the older math module — same philosophy applies here. Word count and output formatting must match the spec exactly or the automated comparisons will fail.
