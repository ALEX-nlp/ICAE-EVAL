## Product Requirement Document

Hey team, we need to wrap up the markdown rendering library integration. The core engine is mostly there but we need a proper adapter that reads JSON from stdin and writes results to stdout — basically the same pattern we used on that content pipeline project last quarter, you know the one. The adapter needs to handle all the rendering modes we've talked about: the default HTML output, that normalized re-serialization mode, and the XML tree dump. Extensions should be toggleable per-request too — things like tables, checkboxes on list items, the strikethrough/superscript stuff, auto-detected links, footnotes, that anchor-on-headings thing, and definition lists. There's also some safety stuff around raw HTML passthrough that the security team flagged — by default it should be suppressed, but callers can opt in. Oh and there's a tag filtering layer on top of that when unsafe mode is on. Smart punctuation (the curly quotes / em-dash thing) lives on the parse side not the render side, just FYI. Bullet marker style and line-wrap width are configurable for the normalized output mode. If anything goes wrong parsing the request or running the render, we should still return a well-formed JSON error rather than crashing. The whole thing needs to stay decoupled — the business logic shouldn't know anything about stdin/stdout.
