## Product Requirement Document

Hey team, we need to wrap up the contract testing adapter work. The core pieces are mostly spec'd out but there are a bunch of edge cases and default behaviors that aren't fully nailed down in the ticket. A few things that have been biting users lately: people keep getting confused about what happens when they pass in a minimum array size of zero or a negative number — we need a clear error there. Also some folks on Windows are hitting issues because we're not picking the right executable name for their platform (remember how we handled that in the platform detection module we did for the other CLI tool? do something similar here). The mock server startup args need to be exactly right — the log file path construction especially trips people up. For the verifier CLI, if someone only gives one of the two provider-state config options instead of both, that should be caught early. There's also the question of what the default working directory is for logs and pact files — I think we agreed it should be the testbed path but I don't see it written down anywhere. Oh and for HTTPS connections, cert and key paths should show up in the process command. Make sure the broker password gets pulled from the environment, not passed directly. Timeout default for the verifier should also be something sensible. Let me know if you have questions, I'll be around.
