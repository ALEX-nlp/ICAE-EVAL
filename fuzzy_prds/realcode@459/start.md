## Product Requirement Document

We need a report generator that turns structured test-run data into a small set of browsable HTML pages for developers and CI dashboards. The input is a set of test class records, each containing the class name, overall counts (total tests run, errors, failures, skipped), and a list of individual test cases with their name, duration, and outcome text. The generator should produce two kinds of pages: a summary index that lists every class in a table with its counts and a hyperlink to that class's detail page, and a per-class detail page that lists every individual test case with duration and outcome. Both pages should use a shared visual structure — a standard HTML5 document with a linked stylesheet — and both should pretty-print the markup with two-space indentation for readability. Lines must end with carriage-return-plus-linefeed so the output is consistent across operating systems. Rows in both tables should carry a style hint so a stylesheet can color-code them: a row where everything passed gets tagged as successful, a row with at least one failure gets tagged as failed, and a row with errors or skips but no failures gets no tag at all. The detail page also includes a back link to the summary. The stylesheet reference should incorporate the configured output directory path. Count fields that cannot be parsed as integers should silently fall back to zero rather than causing an error. When the class list is empty the table still appears, just with only the header row.
