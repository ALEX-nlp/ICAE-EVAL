## Product Requirement Document

We need a small collection of low-level binary encoding utilities that a storage engine can rely on as its foundation layer. The goal is to have one agreed-upon, pure-function implementation for each primitive so that different parts of the system are never inventing their own byte-packing schemes and silently disagreeing. The primitives we need are: a compact variable-length encoding for non-negative integers (both 32-bit and 64-bit variants) that packs small values into fewer bytes; two fixed-width encodings (an eight-byte slot and a six-byte slot) for values that need to live at known offsets; conversion between raw bytes and a human-readable hexadecimal representation using uppercase letters; a lightweight checksum-tagging scheme that stamps a parity signature into the low bits of a value (supporting different tag widths) and later validates and strips it — with a clear failure signal when the checksum is broken rather than silently returning garbage; and a set of bit-arithmetic helpers (rounding up to the next power of two, finding the set-bit index of a power of two, building a low-bit mask, and zero-extending a 32-bit value into 64 bits). All of these should be pure transformations with no shared state or I/O. The hex format must use uppercase digits. The fixed-width long codec must preserve the full signed range including negative values. The six-byte codec is for non-negative values only. The parity tagging must work for tag widths of one, three, four, and sixteen bits.
