## Product Requirement Document

We need a lightweight, reusable toolkit of pure helper functions for cleaning and interpreting web content without requiring network access. The library should cover several categories of operations: normalizing messy page text (collapsing extra whitespace, stripping trailing site-name suffixes from titles), classifying web addresses by the type of content they likely point to (video, audio, downloadable documents, archives, executables, or readable pages), and cleaning up URLs by following well-known link-forwarding interstitials from major platforms and stripping campaign analytics tags from query strings. The toolkit should also support extracting metadata from page markup fragments — specifically parsing icon size declarations to find the best high-resolution favicon, and pulling the single most representative image or link reference out of an arbitrary HTML subtree. All operations must be pure and deterministic given the same input, with no side effects or external calls. The I/O contract is a simple line-oriented key-value protocol where each result is emitted as one or more lines in the form "key=value", with booleans written as lowercase words. URL results must be returned in normalized form (a bare host URL gains a trailing slash). When classifying a URL, the system must output exactly seven boolean lines in a fixed order. The image extractor searches for content in a specific priority order — checking direct attributes on the root element before descending into the subtree and finally inspecting inline style declarations for background references.
