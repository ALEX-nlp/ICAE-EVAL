## Product Requirement Document

We need a front-end component that takes raw JavaScript or TypeScript source text and tells us what happened when we tried to understand it. The caller passes a chunk of code and a few flags about what dialect it might be — whether it uses type annotations, whether it might be a declaration file, whether it has JSX syntax, and whether it should be treated as a module or a classic script. The component then hands back a structured summary: did the parse completely fall over, how many top-level constructs are in the program, how many of those leading string literals count as directives (like 'use strict'), how many comments were pulled out as trivia, what kind each comment is, how many problems were found, and what the problem messages say. If the dialect is something we don't support — handle it the same way the rest of the diagnostics pipeline handles unsupported features — with a clear stable message rather than producing misleading output. The output should be a fixed line-oriented report so any downstream tool can read it without parsing JSON.
