## Product Requirement Document

We need a thin decorator layer that wraps any existing database driver and produces a complete, human-readable audit log of every database call that passes through it. When a write statement is forwarded, the log should record the operation type and the SQL text, and if any parameter values were bound it should also record those values in binding order. When a query is issued it should be logged the same way. Transaction boundaries should be clearly demarcated: the start of a transaction, every statement that runs inside it, and whether it ends with a commit or a rollback. The log entries should come out in the exact order that operations execute, so the output can be used to assert on the complete call sequence in tests. The decorator shouldn't do any of the actual database work itself — it should forward everything to the underlying driver unchanged and just observe what goes by. Bound parameter values should be rendered as a bracketed, comma-separated list. Empty binder lists (explicitly supplied but with nothing bound) should not produce a value log entry — only omitted binder fields skip the binding step entirely, while an empty list still runs the step but produces no output since nothing was bound.
