## Product Requirement Document

We need a configurable text-conversion engine that turns Markdown source into clean HTML fragments. Authors write content in plain-text shorthand — headings, lists, emphasis, code blocks, links, images, tables, and so on — and the engine produces the corresponding HTML. The active set of constructs should be configurable per call: a caller can say 'only enable these features', 'add these on top of the defaults', or 'turn these off', so it's possible to restrict what gets processed for a given document — for example, let only headings and paragraphs through, or disable raw HTML passthrough for untrusted input. One special flag controls whether inline markup inside headings gets processed at all. The default set includes all the common constructs but deliberately excludes raw HTML passthrough and the math block, which must be explicitly turned on. When a paragraph contains multiple lines of text they should be joined with a single space and the block should end with a trailing space before its closing tag — handle that consistently the way the rest of the block rules handle running text.

One small follow-up from the questions that came in: for plain running text, a paragraph keeps a single trailing space right before </p>, and if the text for that one paragraph spans multiple lines, those adjacent lines get joined with a single space. So 'line1\nline2\n' produces '<p>line1 line2 </p>'.

Also confirming the defaults and names exactly: raw HTML passthrough ('html') and the math/latex block ('latex_block') are OFF by default. All other constructs are on by default. Valid feature names are: break_line, checklist, code_block, emphasis, headline, horizontal_line, html, image, inline_code, italic, link, ordered_list, paragraph, quote, strikethrough, strong, table, unordered_list, latex_block.

And for checklist rendering, the outer list is still a <ul> but it needs class="checklist". Each item should be wrapped in <label>...</label>. An unchecked item produces <input type="checkbox"/>. A checked item (bracket with 'x') produces <input type="checkbox" checked="checked"/>.
