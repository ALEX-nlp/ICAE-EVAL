## Product Requirement Document

We need a standalone statistics computation service that developers can call from the command line. The tool should accept a single instruction over standard input and return the result to standard output. At a high level, it needs to support three categories of functionality: summarizing collections of numbers (like finding the center and spread of a dataset), evaluating various probability models for both discrete and continuous scenarios, and running standard hypothesis tests to help decide whether data supports a given assumption.

For probability models, we need to cover the common shapes — things like coin-flip style models, count-based models, bell-curve models, and several others. Each model should expose the typical properties developers ask about (densities, cumulative probabilities, means, modes, etc.).

For the hypothesis tests, we want at least four or five standard tests — the kind you'd find in an intro statistics textbook. Results should be reported as structured key-value output so they're easy to parse.

The tool should handle edge cases gracefully: invalid inputs should return a designated sentinel value rather than crashing, and errors should be surfaced as a readable error message rather than a stack trace. For output formatting, handle it the same way the existing rank-correlation feature does when reporting per-item detail. The optional rounding parameter behavior should match what the chi-squared test module does for its composite output.

All numeric results should use the platform's default floating-point representation, and non-finite values should be rendered as human-readable words rather than raw symbols.

A couple small follow-ups from the questions that came in. When something doesn’t have a defined result, the program should print the exact string 'nil' followed by a newline and stop there. That covers examples like factorial of a negative number, bernoulli PMF outside {0,1}, and beta mode when alpha<=1 or beta<=1. Along the same lines, non-finite float results should be printed as the literal strings 'Infinity', '-Infinity', or 'NaN', and specifically not 'Float::INFINITY' or numeric symbols. So if t variance lands in the case where 1 < df <= 2, that should come out as 'Infinity\n', and inverse standard normal at value=0 should come out as '-Infinity\n'.

On the test decision flags, the rule is strict on one side and inclusive on the other: keep the null hypothesis with null=true when alpha < p_value, and accept the alternative with alternative=true when p_value <= alpha. We want that used the same way in chi_squared_test, f_test, ks_test, t_test, and rank_sum test. For t_test specifically, with tails='two_tail', p_value = 2 * (1 - t_distribution.cumulative_function(abs(t_score))). With tails='one_tail', p_value = 1 - probability where probability = t_distribution.cumulative_function(t_score).

A few function-specific details too. binomial.mode normally returns floor((n+1)*p). If (n+1)*p is a whole number strictly between 1 and n, it should return BOTH adjacent integers as a list '[floor_val, floor_val-1]' with the larger one first. If (n+1)*p equals n+1, return n. The concrete example here is n=11,p=0.5, where (n+1)*p=6, so the result should be '[6.0, 5.0]'. For inverse_standard_normal.cdf, return nil for inputs < 0 or > 1, return '-Infinity' for input exactly 0, and return 'Infinity' for input exactly 1. Inputs like -1e-7 and 1.0000001 should yield nil.

For chi_squared_test.statistic, the degrees of freedom are (number of observed buckets - 1), and expected can be either a single number used for every bucket or a per-bucket list. The output there is an array [chi_statistic, degrees_of_freedom]. For f_test.anova_score, if there are exactly two groups, use F = larger_variance / smaller_variance with numerator df = 1 and denominator df = N-2, where N-2 means total elements minus 2. If there are more than two groups, use the between/within variance decomposition, with numerator df = (groups-1) and denominator df = (total_elements - groups). That should return '[F, df1, df2]'.

Also, collection.variance should be the SAMPLE variance, so divide by (n-1), not n. The formula is sum((x_i - mean)^2) / (n-1), and for [1,2,3,4,5] the answer is 2.5, not 2.0. math.factorial should return nil for n < 0, return 1 for n=0 or n=1, and for larger values it should truncate any fractional part first with n.to_i before computing. It uses Math.gamma(n+1) internally.

And one formatting note since this came up again: rank_correlation.ranks_detail should print each distinct value as 'value={counter: N, rank: R, tie_rank: T}' on its own line, one line per value, in original insertion order. That’s the same behavior implemented in RcbFormat.top (Hash branch) and RcbFormat.inline (Hash branch) in bin/dispatcher.rb and golden/lib/statistics/spearman_rank_coefficient.rb.
