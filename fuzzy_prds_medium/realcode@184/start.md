## Product Requirement Document

Our API endpoints need to serialize domain objects to JSON in a controlled way. Right now every endpoint hand-crafts its own JSON, leading to inconsistent field selection, key naming, and nesting. We need a declarative serialization engine where a developer describes what the output should look like and the engine does the transformation. The spec should support: choosing exactly which fields appear and in what order; optionally wrapping the output under a root key (with separate keys for single vs collection); rendering a collection as either a plain array or as a lookup object keyed by a chosen field; embedding related records and inline groupings at arbitrary nesting levels with optional output key renaming; transforming key casing to match the naming convention of the API consumer (camel, lower-camel, dash, snake); conditionally including a field based on a predicate checked against the record value or request parameters; substituting a fixed placeholder when a field value is null; silently dropping or nullifying fields that fail to resolve; and attaching metadata like a count. Runtime overrides (a root key supplied at request time) should take precedence over the spec declaration. Key transformation must be optionally cascaded into nested associations — look at how the existing 'strict' transformation mode is wired in the transformation pipeline to understand whether cascade is the default.

One small follow-up from the questions that came in: key transformation cascade should default to false. In other words, unless cascade is explicitly set to true, a parent’s key transformation does NOT propagate to nested associations, and the nested association keeps its original field names. Also confirming the runtime behavior on wrapping: yes, a root_key supplied at serialization call time overrides any root_key declared in the resource specification, and that runtime key wraps both singular and collection payloads.

Also on null handling, when on_nil is set to a constant value, every attribute whose resolved value is null is replaced by that constant in the output. The constant is emitted as-is in the JSON.
