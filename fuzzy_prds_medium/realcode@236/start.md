## Product Requirement Document

We need a small utility toolkit for our desktop application that solves two recurring pain points. First, our app needs to remember and restore window positions between sessions. We store everything as plain text in a simple settings file, so we need a reliable way to convert a screen region — defined by its origin corner and its size dimensions — into a text snapshot and back. Numbers must always appear in their decimal form even when they are whole, so the format is consistent. Second, our app maintains live snapshots of running programs and their on-screen windows. These snapshots must be treated as read-only values — you never mutate one; instead you produce a modified copy where only the fields you care about change and everything else stays the same. Two snapshots should be considered identical whenever their contents match, regardless of whether they are the same object in memory. The process snapshot holds an identifier name, a numeric handle, and a lifecycle state chosen from three possible states (one normal, one paused, one indeterminate). The window snapshot wraps a process snapshot and adds a numeric window handle and a display label. For the lifecycle state field, please look up what values the existing domain model already recognises so nothing is invented from scratch. The copy-with-overrides pattern in this codebase follows a specific convention — check the relevant model file to understand exactly how unset optional parameters fall back to the original value.

One small clarification on the status field: the ProcessStatus enum defines exactly three values: normal, suspended, unknown. Those are the only permitted status values, and in Dart they’re referenced by name like ProcessStatus.normal. The existing enum lives in golden/lib/native_platform/src/process/models/process.dart, so please use that as the source of truth rather than introducing anything new.

Also, on the rectangle text format, this is specifically a Flutter Rect created via Rect.fromLTWH(left, top, width, height) where left/top are the origin and width/height are the extent. All four values left, top, width, height need to be emitted in floating-point / decimal form every time, so a whole number like 100 must be written as 100.0, not 100. The encoder uses Dart's Rect.toJson() which serializes via jsonEncode, producing for example {"left":0.0,"top":0.0,"width":100.0,"height":100.0}. The serialized rectangle JSON object must list the four fields in exactly this order: left, top, width, height, because RectConverter.toJson() builds the map in that order. The output line ends with a trailing newline.

And for the read-only snapshot copying behavior, the fallback convention is the usual Dart null-aware one. Process.copyWith() has optional nullable params and uses the pattern `executable: executable ?? this.executable`, so if a parameter is not supplied (null), the original field value is preserved. Same idea for Window.copyWith() for id, process, and title. If you copy with no overrides, you should get a record that is value-equal to the original (equals_base=true).
