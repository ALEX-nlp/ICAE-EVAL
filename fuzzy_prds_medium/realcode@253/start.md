## Product Requirement Document

We need a library for working with the SAML2 single sign-on protocol at the level of well-typed domain objects. Integration teams working with identity providers and service providers currently hand-write fragile XML parsing, mishandle time zones in timestamps, and re-implement the URL encoding that carries protocol messages across the browser. We want one well-defined contract covering: a UTC timestamp value type that only accepts properly UTC-tagged instants (with one exception for the zero/uninitialized default), comparison of offset-aware timestamps against UTC ones; a name-identifier type whose equality rules match the SAML spec (two identifiers are equal only when both their value and their optional format agree, an absent format only matches an absent format); readers that parse authentication request XML, assertion XML, protocol response XML, entity-descriptor metadata XML, and identity-provider role descriptor XML into domain objects; and a redirect-binding decoder that recovers a message from a query-string URL. All structural failures should surface as stable named error categories — handle the error reporting the same way as the XML traversal layer does, never surfacing host-language runtime details or stack traces. The redirect binding uses raw-deflate compression and URL-safe base64. Metadata entity descriptors must validate that the root element is in the correct SAML namespace and that at least one role descriptor child is present.

One small clarification on the timestamp behavior since the team asked about the default case: the zero/default timestamp (ticks=0) is accepted when its kind marker is 'unspecified'. This represents the uninitialized default. All other unspecified-kind timestamps are rejected with error=non_utc_datetime_kind. 'local' kind is always rejected regardless of tick count.

Also wanted to pin down the stable error names so nobody has to guess later. For XML shape issues, use 'missing_attribute' when a required XML attribute is absent, 'missing_element' when a required child element is absent, 'unexpected_local_name' when an element has the wrong local name, and 'unexpected_namespace' when an element is in the wrong XML namespace. On the redirect side, use 'message_parameter_not_found' when there is no SAMLRequest or SAMLResponse, 'duplicate_message_parameters' when both SAMLRequest and SAMLResponse are present, and 'duplicate_relay_state' when RelayState appears more than once.

And on the browser redirect encoding, the XML is raw-deflate compressed (no zlib header, RFC 1951), then URL-safe base64 encoded. The query parameter is named exactly 'SAMLRequest' or 'SAMLResponse'. An optional 'RelayState' parameter carries metadata verbatim.
