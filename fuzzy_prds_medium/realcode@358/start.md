## Product Requirement Document

We need the platform-agnostic contract layer for our location services, sitting between the app and whatever operating-system provider it runs on. The library needs four pieces: First, coordinate math — computing the straight-line surface distance between two GPS points and the initial heading from one to the other. The distance must be zero when both points are identical and symmetric when start/end are swapped. Bearing uses a signed convention where east-of-north is positive and west-of-north is negative. Second, the location reading value type — deserialized from the raw map the OS hands back. Latitude and longitude are mandatory; everything else has a default. Treat missing numeric fields the same way our sensor-data normalizer handles absent values. When a mandatory field is missing, fail fast with a neutral error line naming the missing field. Third, permission and accuracy catalogs — fixed ordered sets whose zero-based position in the list is the integer code exchanged with the platform. Codes 0-3 map to permission statuses; code 4 is a special 'unable to determine' status. Any other code is invalid. Fourth, domain errors — several categories accept an optional custom message (fall back to a default when none provided), two have a fixed description that never changes.

One small follow-up on the catalogs and the location value defaults, since the team asked how literal we want to be here. For permissions, the ordered mapping is 0=denied, 1=deniedForever, 2=whileInUse, 3=always. Code 4 is unableToDetermine in the catalog, but code 4 is invalid for permission_from_code since that operation only maps 0-3. Anything outside 0-3 should come back exactly as error=invalid_permission\nvalue=<n>.

For accuracy, the ordered set is 0=lowest, 1=low, 2=medium, 3=high, 4=best, 5=bestForNavigation, 6=reduced. Count=7.

And on the deserialized location reading, all numeric optional fields default to 0.0: accuracy, altitude, altitude_accuracy, heading, heading_accuracy, speed, speed_accuracy. floor defaults to null (absent/unset). is_mocked defaults to false.
