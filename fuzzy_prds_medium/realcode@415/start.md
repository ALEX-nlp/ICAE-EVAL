## Product Requirement Document

Our authorization server needs a shared set of small, pure helper functions for deciding whether a request is permitted. These cover several concerns: (1) scope matching — given the set of scopes a client was actually granted, decide whether a requested scope is covered, with support for three matching strategies: one that treats scopes as a dot-separated hierarchy where broader scopes cover narrower ones; one that supports wildcard segments; and one that requires an exact match; (2) set and sequence comparisons — unordered case-insensitive equality, ordered case-sensitive equality, contains-all, contains-any, single-value equality, and space-joined equality for comparing lists of permission arguments; (3) two low-level string helpers: case-insensitive membership check and a minimal escaper for embedding strings safely in JSON; (4) audience checks — URI-aware matching (same scheme, host, and path prefix) and exact-string matching; and (5) redirect-URI validation — loopback/localhost detection, transport-security checks in standard and strict modes, and matching a requested URI against a registered set. Additionally we need a utility to append a key=value pair to the fragment component of a URL. All functions are pure, take plain inputs, and return plain outputs — no I/O or global state. Follow the same loopback-detection logic as the existing token-endpoint host check.

A couple follow-ups from the team’s questions. On wildcard scope matching, if the pattern has fewer segments than the request, its FINAL segment must be '*' to match the extra segments. A non-'*' final segment only matches when the pattern and request have the same segment count. Also, an empty request segment is never matched by '*'.

On unordered case-insensitive set equality in Matches, probe items that collapse to the same case-folded value effectively reduce the set size below the reference length, causing a length mismatch and returning false. Example: reference=['foo','foo'], probe=['foo','foo'] returns false (both probe items are the same case-folded value).

Also, for the localhost / loopback piece, a URL is treated as localhost when its host is exactly 'localhost', '127.0.0.1', or '::1', OR when the host ends with the suffix '.localhost'. Port is ignored for this check.
