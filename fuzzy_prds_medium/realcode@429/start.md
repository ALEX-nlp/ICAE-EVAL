## Product Requirement Document

We need a small, self-contained utility library for an e-book reading engine that handles three categories of low-level string operations. First, the library must be able to look at a resource reference and tell the caller whether that reference points inside the publication bundle or out to the internet. An empty string should be treated as an internal reference, and passing nothing at all should raise an error in the same manner as the authentication module does for missing tokens. Second, the library must be able to extract the directory portion from a file path, and also combine a base directory with a relative file reference while correctly resolving any parent-directory traversal segments; both operations must tolerate messy duplicate slashes by normalizing them away. Third, the library must split a hyperlink reference into its resource address and its optional in-page anchor, keeping track of whether an anchor was present at all (so a missing anchor can be distinguished from an empty one). Finally, the library must parse narration timing values used in synchronized audio playback. Timing values come in two flavors: one uses colon-separated fields for hours, minutes, and seconds, and the other uses a plain number followed by a metric unit suffix. Both flavors must be handled by a single parser that normalizes the result into hour, minute, second, and millisecond components and produces a canonical string rendering in the same format as the timestamp formatting used elsewhere in the narration subsystem. Any malformed input should cause the parser to signal failure rather than throw an exception.

One small follow-up from the questions that came back: for the local vs external check, the rule is intentionally very literal. A path is non-local if and only if it contains the substring '://' anywhere in the string. If it does not contain '://', it is local, including an empty string. If the path argument is null, it throws ArgumentNullException with param name 'path', and the dispatcher should surface that as error=argument_null / param=path. Same general pattern applies anywhere this utility is supposed to reject a missing required argument: Throws ArgumentNullException with the parameter name of the offending argument, which the dispatcher catches and renders as error=argument_null followed by param=<argument_name>. See ContentPathUtils.IsLocalPath and UrlParser constructor for the pattern.

Also, one edge case on Combine: when both inputs are null, or both resolve to null/empty, the function returns null internally. In that case the dispatcher does not make up a path string, it renders the null result as the literal token '<null>', so the output is exactly combined=<null>.

For narration times, the canonical output format is h:mm:ss[.fff]. That means the hour is unpadded, minute and second are always two digits, and the fractional part '.fff' is included only when the millisecond value is non-zero. The examples to line up with are 2:30:03, 0:00:10.500, and 0:00:00.005, including the cases called out as ms=500 and ms=5. On the unit-style inputs, the only valid suffixes are exactly 'h', 'min', 's', and 'ms'. They are case-sensitive and must be lowercase. 'H', 'Min', 'S', 'MS', 'Sec' and any other casing should fail. If there is no unit at all, treat the number as seconds. There is also a hard upper bound for timecount magnitude: (Int64)Int32.MaxValue * MILLISECONDS_IN_HOUR, which equals (long)2147483647 * 3600000 = 7730941132800000000 milliseconds. Anything larger than that should make TryParse return false rather than throw.
