## Product Requirement Document

We need a translation preparation toolkit for our content management system. When editors write rich text — paragraphs, headings, lists, links, and formatting — we need to pull out just the translatable pieces so translators never see HTML infrastructure or link URLs they could accidentally break. The key steps are: first, escape any special characters in plain text so they're safe to embed in markup; second, strip technical attributes off inline tags and save them in a side map so translators only touch clean markup; third, after translation comes back, check the result still has no stray attributes (except the identifiers we put in); fourth, render a translated piece back to plain text or restore it to full HTML by plugging the saved attributes back in by their identifiers (not by position). We also need to handle full document extraction — pulling out each paragraph, heading, or list item as a separate segment — and later put translated segments back into the original document structure. Handle the identifier tag numbering the same way as the attribute-lifting step does (a-tags get a1, a2, etc. in document order). Finally, we need link-integrity checks: after translation, no new link identifiers should appear that weren't in the source.

One extra bit the team asked about: for inline strings, the only tags we should allow are a, abbr, acronym, b, code, em, i, strong, br. Things like p, h1, ul, li, img, etc. are not allowed to show up inside that kind of string, and if we hit one we should emit exactly 'error=disallowed_tag\ntag=<tagname>'. On the id side, numbering is per tag name in document order, so the first anchor is 'a1', the second is 'a2', and each tag name keeps its own separate counter. Also, if an anchor has no attributes, it should not get a generated id. The output format here should stay as 'string=<normalized>' and then 'attrs=<json_object>'.

Also clarifying the translated-fragment validation piece: in validate_translated, NO attributes are allowed except an 'id' attribute on an 'a' tag. If anything else shows up as an attribute, that should trigger 'error=disallowed_attributes'. And same as above, if a block-level tag appears in the translated fragment, that should come back as 'error=disallowed_tag' with 'tag=<name>'.
